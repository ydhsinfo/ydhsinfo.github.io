프로젝트 활동

할머니를 지켜 줄 위험 방지 장치

가속도 센서와 라디오 통신을 활용하여 위험 상황을 감지하고 알리는 IoT 시스템을 만들어 봅시다

융합(통합 사회)
소요 시간: 약 50분
마이크로비트 2대

문제 상황

프로젝트의 출발점

👵

송이 할머니

1

송이 할머니는 혼자 생활하신다.

2

가족은 할머니가 혼자 계시다가 발생할 위험 상황을 걱정한다.

?

할머니에게 발생한 위험 상황을 어떻게 알 수 있을까?

1

단계 1: 문제 정의하기

문제 분석 및 정의

할머니의 활동이 일정 시간 멈춘 것을 감지하여 위험 상황을 다른 사람에게 전송하는 시스템을 만들자.

입출력 설계

입력 처리 출력
가속도 센서값
일정 시간 이상 가속도 센서값의 변화가 없으면
라디오 신호를 송출
가족의 장치에
'danger'를 출력
2

단계 2: 장치 구성하기

하드웨어 준비 및 구성

준비물

화장실 문 부착 장치

마이크로비트 1개

  • 라디오 신호 장치 (내장)
  • 가속도 센서 (내장)

수신 장치

마이크로비트 1개

  • 라디오 신호 장치 (내장)
  • LED (내장)
참고 — 화장실은 반드시 가야 하는 장소이므로, 오랫동안 문을 여닫지 않는 것을 이상 징조로 판단합니다.
실습에서는 10초로 설정하여 테스트합니다.
3

단계 3: 알고리즘 설계하기

논리 흐름 설계

화장실 문 부착 장치 알고리즘

1

라디오 그룹 1 설정 (모둠별 다르게)

2

변수 a에 가속도 센서값 저장 → 100ms 후 변수 b에 가속도 센서값 저장
(x축값만 측정, 범위: -1024 ~ +1024)

3

a - b 값이 100보다 작으면 움직임 없다고 판단 → count 1 증가

4

count값이 100(10초)보다 크면 → 라디오 신호로 'danger' 송출

수신 장치 알고리즘

1

라디오 그룹 1 설정 (모둠별로 다름)

2

수신한 라디오 신호를 LED에 출력

3

버튼을 눌러 메시지 삭제

4

단계 4: 프로그래밍하기

블록 코딩 구현

(1) 화장실 문 부착 장치 프로그램

시작하면
라디오 그룹 1 // 설정
count = 0 // 초기화
무한반복
a = 가속도 센서 x축 값
일시 중지 100ms
b = 가속도 센서 x축 값
만약 100 > 절대값(a - b) 이면
count += 1 // 움직임 없음 카운트
아니면
count = 0 // 움직임 감지, 초기화
만약 count > 100 이면
라디오 전송 'danger'
count = 0 // 송출 후 초기화

(2) 수신 장치 프로그램

시작하면
라디오 그룹 1 // 설정
라디오 문자열 수신 시
문자열 출력 receivedString // 받은 문자열 LED에 표시
A 버튼 누를 때
LED 스크린 지우기 // 화면 초기화
5

단계 5: 실행하기

테스트 및 검증

움직임 감지 시

화장실 문 부착 장치를 가진 사람이 10초 안에 한 번이라도 마이크로비트를 흔들면 수신기는 변함이 없습니다.

움직임 미감지 시

10초 이상 방치하면 수신기에 'danger'가 스크롤됩니다.

생각해 보기

10초의 움직임 감지 시간을 5시간으로 늘리려면 count값을 얼마로 설정해야 할까요?

힌트: 100ms마다 count가 1씩 증가합니다. 10초 = 100 count이면, 5시간 = ? count 인지 비례식으로 계산해 보세요.

실생활 활용

IoT 기술의 확장

움직임 감지 활용

가속도 센서, 동작 센서를 이용하면 일정 시간 이상 움직임이 없을 때 신호를 보내 위험 상황을 알릴 수 있습니다.

침입 감지 활용

반대로 움직임을 감지할 때 신호를 보내서 외부 침입자를 감지하는 용도로도 활용할 수 있습니다.
