네트워크 환경

네트워크 환경을 구성하여 인터넷을 연결하고, 다양한 자원을 공유하는 방법을 알아봅시다

고등학교 정보 교과서
학습 시간: 약 50분
2개 핵심 주제

학습 목표

  • 네트워크 환경을 구성하여 인터넷에 연결할 수 있다.
  • 네트워크를 이용하여 다양한 자원을 공유할 수 있다.
  • 클라우드 컴퓨팅의 개념을 이해하고 활용 사례를 설명할 수 있다.
1

인터넷 네트워크의 연결

컴퓨팅 시스템으로 인터넷을 사용하려면 랜(LAN) 카드에 선을 직접 연결하거나, 와이파이(Wi-Fi)로 무선 연결한 후 운영체제에서 네트워크 환경을 설정해야 합니다. 운영체제의 네트워크 관련 설정에서 IP 주소를 확인하거나, 접속 가능한 와이파이를 확인할 수 있습니다.

(1) 내 컴퓨터의 네트워크 정보 알아보기

Windows 운영체제에서 네트워크 정보를 확인하는 방법을 단계별로 알아봅시다.

1

Windows 설정 열기

시작 메뉴에서 설정을 클릭하거나, 키보드에서 Win + I 단축키를 사용합니다.

2

네트워크 및 인터넷 선택

설정 화면에서 네트워크 및 인터넷 항목을 클릭합니다.

3

이더넷 속성 확인

왼쪽 메뉴에서 이더넷을 선택한 후, 속성 버튼을 클릭합니다.

4

네트워크 정보 확인

속성 창에서 다음과 같은 네트워크 정보를 확인할 수 있습니다:

IPv4 주소

192.168.0.xxx

서브넷 마스크

255.255.255.0

기본 게이트웨이

192.168.0.1

DNS 서버

8.8.8.8

(2) 컴퓨터에 와이파이 연결하기

Windows에서 와이파이(Wi-Fi) 네트워크에 연결하는 방법을 알아봅시다.

1

Wi-Fi 아이콘 클릭

작업 표시줄 오른쪽 하단의 Wi-Fi 아이콘을 클릭하여 네트워크 목록을 엽니다.

2

네트워크 선택

사용 가능한 Wi-Fi 목록에서 연결할 네트워크를 선택합니다.

3

비밀번호 입력

보안이 설정된 네트워크는 비밀번호를 입력한 후 연결 버튼을 클릭합니다.

팁: '자동으로 연결' 옵션을 체크하면, 해당 Wi-Fi 범위 안에 있을 때 자동으로 연결됩니다. 학교나 집에서 자주 사용하는 네트워크는 자동 연결을 설정해 두면 편리합니다.

IP 주소란?

IP 주소(Internet Protocol Address)는 인터넷에서 사용하는 주소로, 네트워크에 연결된 각 장치를 식별하기 위해 부여되는 고유한 번호입니다. 인터넷 주소 체계로 IPv4와 IPv6가 사용됩니다.

IPv4

32비트로 구성된 주소 체계

192.168.0.1

약 43억 개의 주소 표현 가능

IPv6

128비트로 구성된 주소 체계

2001:0db8:85a3::8a2e:0370:7334

사실상 무한한 주소 표현 가능

내 IP 주소를 쉽게 확인하는 방법

윈도우 검색창이나 웹 브라우저에서 'What is my ip'을 검색하면 현재 사용 중인 공인 IP 주소를 바로 확인할 수 있습니다.

명령 프롬프트(cmd)를 열고 ipconfig 명령어를 입력합니다.

C:\Users\User> ipconfig
Windows IP 구성
이더넷 어댑터 이더넷:
IPv4 주소 . . . . . . . : 192.168.0.10
서브넷 마스크 . . . . . : 255.255.255.0
기본 게이트웨이 . . . . : 192.168.0.1

네트워크에서의 공유

네트워크를 사용하면 떨어져 있는 다양한 컴퓨터 시스템의 문서, 사진 등 데이터에 접근할 수 있습니다. 또한 프린터나 파일 서버 등 연결된 장치에도 접근할 수 있습니다.

네트워크 공유의 개념

(1) 개인용 컴퓨터에서의 공유

파일 공유

공유 설정으로 지정된 폴더의 파일을 네트워크에 연결된 다른 컴퓨터와 공유할 수 있습니다. 같은 네트워크에 있는 사용자는 공유 폴더에 접근하여 파일을 열거나 복사할 수 있습니다.

Windows에서는 폴더를 마우스 오른쪽 클릭 후 [속성] > [공유] 탭에서 설정할 수 있습니다.

프린터 공유

내 컴퓨터에 연결된 프린터를 네트워크를 통해 다른 사람이 사용할 수 있도록 공유할 수 있습니다. 한 대의 프린터에 여러 컴퓨팅 시스템을 연결하여 효율적으로 사용할 수 있습니다.

[설정] > [장치] > [프린터 및 스캐너]에서 프린터를 공유할 수 있습니다.

(2) 클라우드 컴퓨팅을 활용한 공유

클라우드 컴퓨팅을 사용하면 서버에서 동작하는 소프트웨어를 사용하거나, 서버에 데이터를 저장하고 사용하는 등 다양한 공유 방법을 활용할 수 있습니다. 장소에 상관없이 여러 사람이 함께 문서를 작성하는 것도 가능합니다.

클라우드 컴퓨팅이란?

클라우드 컴퓨팅(Cloud Computing)이란 사용자들이 시간과 장소에 상관없이 네트워크로 연결된 컴퓨팅 자원을 사용할 수 있는 시스템입니다. 저장 장소를 통해 데이터를 사용하고, 서버의 다양한 소프트웨어를 사용할 수 있습니다.

핵심 정리

1

인터넷을 사용하려면 랜 카드에 유선 연결하거나 와이파이로 무선 연결한 후, 운영체제에서 네트워크 환경을 설정해야 합니다.

2

IP 주소는 인터넷에서 사용하는 고유한 주소로, IPv4와 IPv6 체계가 있으며 설정 또는 명령 프롬프트에서 확인할 수 있습니다.

3

네트워크를 통해 파일 공유, 프린터 공유 등 다양한 자원을 공유할 수 있습니다.

4

클라우드 컴퓨팅을 활용하면 시간과 장소에 상관없이 소프트웨어를 사용하고 데이터를 저장, 공유할 수 있습니다.

IP 주소와 MAC 주소

추가 학습

IP 주소 (IP Address)

Internet Protocol Address

인터넷에서 컴퓨터나 기기를 식별하는 논리적 주소입니다. 우편 주소처럼 데이터가 올바른 목적지에 전달될 수 있도록 합니다. 네트워크 관리자나 ISP(인터넷 서비스 제공자)에 의해 할당됩니다.

IPv4

32비트 구성, 약 43억 개의 주소

0~255 범위의 숫자 4개를 점(.)으로 구분

192.168.0.1

예시

주소 고갈 문제로 IPv6로 전환 중

IPv6

128비트 구성, 사실상 무한한 주소

16진수 8개 그룹을 콜론(:)으로 구분

2001:0db8:85a3::8a2e

예시

보안, 속도, 효율성이 향상됨

공인 IP vs 사설 IP

구분 공인 IP 사설 IP
의미 인터넷에서 사용하는 전 세계 유일한 주소 내부 네트워크(가정, 회사)에서만 사용
할당 ISP(KT, SKT 등)에서 부여 공유기(라우터)에서 자동 부여
예시 203.0.113.5 192.168.0.x
정적 IP

항상 동일한 주소 유지. 서버 운영에 적합하지만 관리 비용이 높음

동적 IP

접속 시마다 주소가 변경될 수 있음. DHCP 서버가 자동 할당

MAC 주소 (MAC Address)

Media Access Control Address

네트워크 인터페이스 카드(NIC)에 제조 시 부여되는 고유한 물리적 주소입니다. 전 세계에서 동일한 MAC 주소를 갖는 장치는 존재하지 않으며, 하드웨어에 영구적으로 기록됩니다.

MAC 주소의 구조

00:1A:2B : 3C:4D:5E

제조사 코드 (OUI) 장치 고유 번호

앞 24비트 — OUI (제조사 코드)

IEEE에서 제조사별로 할당. 어떤 회사가 만들었는지 식별 가능

뒤 24비트 — 장치 고유 번호

제조사가 각 장치에 고유하게 부여하는 일련번호

주요 특징

48비트(6바이트)로 구성, 16진수 12자리로 표현

콜론(:) 또는 하이픈(-)으로 2자리씩 구분하여 표기

같은 네트워크 내 통신 시 사용되는 데이터 링크 계층의 주소

원칙적으로 변경 불가하지만 소프트웨어로 임시 변경 가능

IP → MAC: ARP (주소 변환 프로토콜)

Address Resolution Protocol

실제 통신 시 IP 주소만으로는 부족합니다. 같은 네트워크 안에서 데이터를 전달하려면 MAC 주소가 필요합니다. ARP는 IP 주소를 이용해 해당 장치의 MAC 주소를 자동으로 찾아주는 프로토콜입니다.

1. 프로토콜의 정의

프로토콜(Protocol)이란 서로 다른 컴퓨터나 기기들이 네트워크를 통해 올바르게 통신하기 위해 약속한 규칙과 절차의 집합입니다. 언어가 다른 두 사람이 대화하려면 공통 언어가 필요하듯, 기기 간 통신에도 공통 규약이 필요합니다.

프로토콜의 3요소

구문 (Syntax)

데이터의 형식, 부호화, 신호 레벨 등 어떻게 표현할지의 규칙

의미 (Semantics)

각 비트가 의미하는 내용, 오류 제어·흐름 제어 등 무엇을 뜻하는지

타이밍 (Timing)

데이터 전송 속도, 순서 등 언제·어떤 순서로 전송할지의 규칙

프로토콜이 정의하는 것들

데이터 형식
오류 처리
전송 순서
연결 방식
주소 지정
흐름 제어
암호화 방식
전송 속도

2. 대표적인 프로토콜 종류

TCP / IP Transmission Control Protocol / Internet Protocol

인터넷의 근간이 되는 핵심 프로토콜 묶음. IP가 주소를 지정하고 경로를 찾으면, TCP가 데이터를 분할하여 신뢰성 있게 전송하고 오류를 처리합니다.

TCP 특징

  • 연결 지향 (3-way handshake)
  • 데이터 전송 순서 보장
  • 오류 검출 및 재전송
  • 신뢰성 높음, 속도 다소 느림

IP 특징

  • 비연결형, 최선 전달 방식
  • 패킷 경로 지정 (라우팅)
  • IPv4 / IPv6 버전 존재
  • 오류 복구 기능 없음
사용 예: 웹 브라우징, 이메일, 파일 전송 등 데이터 정확성이 중요한 모든 인터넷 통신
HTTP / HTTPS HyperText Transfer Protocol (Secure)

웹 브라우저와 웹 서버 사이에서 HTML 문서, 이미지 등 웹 데이터를 주고받는 프로토콜. HTTPS는 HTTP에 SSL/TLS 암호화를 추가하여 보안을 강화한 버전입니다.

HTTP

포트 80

암호화 없음

HTTPS

포트 443

SSL/TLS 암호화
사용 예: 웹 사이트 접속 (주소창의 http:// 또는 https://)
FTP File Transfer Protocol

클라이언트와 서버 간 파일을 업로드·다운로드하기 위한 프로토콜. 제어 채널(명령)과 데이터 채널(전송)을 분리하여 운영합니다.

제어 포트: 21
데이터 포트: 20
보안 버전: SFTP / FTPS
사용 예: 웹 서버에 파일 업로드, 대용량 파일 배포
DNS Domain Name System

사람이 기억하기 쉬운 도메인 이름을 IP 주소로 변환해주는 프로토콜. 인터넷의 전화번호부 역할을 합니다.

www.google.com
DNS 조회
142.250.196.68
사용 예: 웹 주소 입력 시 자동으로 IP 주소 변환 (포트 53 사용)
SMTP / POP3 / IMAP 이메일 프로토콜

이메일 송신과 수신을 담당하는 프로토콜 모음입니다.

SMTP

이메일 발송 (포트 25 / 587)

클라이언트 → 서버, 서버 → 서버 메일 전송

POP3

이메일 수신 (포트 110)

서버 메일을 로컬에 다운로드 후 서버에서 삭제

IMAP

이메일 수신 및 동기화 (포트 143)

서버에 메일 보존, 여러 기기에서 동기화 가능

UDP User Datagram Protocol

TCP와 달리 연결 설정 없이 빠르게 데이터를 전송하는 프로토콜. 신뢰성보다 속도와 실시간성이 중요한 서비스에 사용됩니다.

구분 TCP UDP
연결 방식 연결형 비연결형
신뢰성 높음 낮음
전송 속도 느림 빠름
사용 예 웹, 이메일, 파일 전송 스트리밍, 게임, VoIP
사용 예: 유튜브 스트리밍, 온라인 게임, 화상통화(Zoom)
프로토콜 역할 포트 사용 예
TCP/IP 인터넷 기본 통신 규약 — 모든 인터넷 통신
HTTP 웹 데이터 전송 80 웹사이트 접속
HTTPS 암호화된 웹 통신 443 인터넷 뱅킹, 쇼핑몰
FTP 파일 전송 20/21 서버 파일 업로드
DNS 도메인 → IP 변환 53 웹 주소 입력 시
SMTP 이메일 발송 25/587 메일 보내기
POP3/IMAP 이메일 수신 110/143 메일 받기
UDP 빠른 비연결형 전송 — 스트리밍, 게임

추가 학습

OSI 모델이란?

OSI (Open Systems Interconnection) 모델은 서로 다른 시스템 간의 통신을 표준화하기 위해 ISO(국제표준화기구)가 1984년에 제정한 7계층 참조 모델입니다. 네트워크 통신 과정을 7단계로 나누어 각 계층이 독립적인 역할을 수행하도록 설계되어, 문제 발생 시 해당 계층만 점검하면 됩니다.

표준화

제조사가 달라도 통신 가능한 공통 규격 제공

장애 진단

문제가 발생한 계층을 빠르게 특정하여 해결

모듈화

각 계층이 독립적으로 개발·교체 가능

7 응용 계층 (Application) HTTP · FTP · DNS · SMTP

사용자와 직접 상호작용하는 계층. 이메일, 파일 전송, 웹 브라우저 등 응용 프로그램에 네트워크 서비스를 제공합니다.

HTTP/HTTPS FTP DNS SMTP Telnet
6 표현 계층 (Presentation) SSL · JPEG · ASCII

데이터의 형식 변환, 암호화, 압축을 담당합니다. 서로 다른 시스템이 같은 형식으로 데이터를 이해할 수 있게 합니다.

SSL/TLS JPEG ASCII MPEG
5 세션 계층 (Session) NetBIOS · RPC

통신 기기 간의 연결(세션)을 생성·유지·종료하는 역할. 동기화 기능으로 데이터 복구 지점을 제공합니다.

NetBIOS RPC SSH
4 전송 계층 (Transport) TCP · UDP

종단 간(End-to-End) 신뢰성 있는 데이터 전송을 담당. 데이터를 세그먼트로 분할하고, 포트 번호로 프로세스를 구분합니다.

TCP UDP
데이터 단위: 세그먼트(TCP) / 데이터그램(UDP)
3 네트워크 계층 (Network) IP · ICMP · ARP

IP 주소 기반 경로 설정(라우팅)과 패킷 전달을 담당. 서로 다른 네트워크 간 데이터를 전송합니다.

IP ICMP ARP 라우터
데이터 단위: 패킷(Packet)
2 데이터링크 계층 (Data Link) Ethernet · Wi-Fi

MAC 주소 기반으로 같은 네트워크 내 기기 간 데이터 전송을 담당. 오류 검출 및 흐름 제어를 수행합니다.

Ethernet Wi-Fi(802.11) 스위치
데이터 단위: 프레임(Frame)
1 물리 계층 (Physical) 케이블 · 허브 · 리피터
