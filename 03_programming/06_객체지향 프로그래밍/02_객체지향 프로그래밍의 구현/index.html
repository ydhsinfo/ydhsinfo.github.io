구현 상속·다형성

객체지향 프로그래밍의 구현

캡슐화, 상속, 다형성을 파이썬으로 직접 구현하며 객체지향 프로그래밍의 핵심 특성을 익힙니다.

전체 단원으로 돌아가기

1 캡슐화 (Encapsulation)

캡슐화란 객체의 속성(데이터)과 메서드(기능)를 하나로 묶고, 외부에서 직접 접근하지 못하도록 은닉(hiding)하는 것입니다. 이를 통해 데이터의 무결성을 보호하고, 내부 구현을 숨겨 복잡성을 줄입니다.

파이썬의 접근 제어

public

name

어디서든 접근 가능한 속성·메서드

protected

_name

클래스 내부와 자식 클래스에서만 접근 권장

private

__name

클래스 내부에서만 접근 가능 (이름 맹글링)

캡슐화 구현 예시

Python 은행 계좌 클래스 
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner          # public
        self._bank = "한국은행"    # protected
        self.__balance = balance    # private

    def deposit(self, amount):     # public 메서드
        if amount > 0:
            self.__balance += amount
            print(f"{amount}원 입금 완료")

    def get_balance(self):         # getter 메서드
        return self.__balance

# 사용
account = BankAccount("홍길동", 10000)
account.deposit(5000)
print(account.get_balance())   # 15000
# print(account.__balance)    # 오류: private 접근 불가

2 상속 (Inheritance)

상속이란 기존 클래스(부모 클래스)의 속성과 메서드를 새로운 클래스(자식 클래스)가 물려받아 재사용하고 확장하는 것입니다. 코드의 재사용성을 높이고 계층 구조를 표현합니다.

상속 계층 구조

부모 클래스 (Parent)
Animal
name, age, speak()
자식 클래스 (Child)
Dog
breed, fetch()
자식 클래스 (Child)
Cat
color, purr()

Dog와 Cat은 Animal의 name, age, speak()를 상속받습니다.

상속 구현 예시

Python 동물 상속 구조 
# 부모 클래스
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def speak(self):
        print(f"{self.name}이(가) 소리를 냅니다.")

    def info(self):
        print(f"이름: {self.name}, 나이: {self.age}살")

# 자식 클래스 - Animal 상속
class Dog(Animal):
    def __init__(self, name, age, breed):
        super().__init__(name, age)  # 부모 생성자 호출
        self.breed = breed

    def fetch(self):
        print(f"{self.name}이(가) 공을 가져옵니다!")

# 사용
dog = Dog("멍멍이", 3, "골든 리트리버")
dog.info()    # 부모 메서드 사용
dog.speak()   # 부모 메서드 사용
dog.fetch()   # 자식 메서드 사용

super() 함수

super()는 자식 클래스에서 부모 클래스의 메서드를 호출할 때 사용합니다.

  • super().__init__() — 부모 클래스의 생성자를 호출하여 속성을 초기화
  • super().메서드() — 부모 클래스의 특정 메서드를 호출

3 다형성과 메서드 오버라이딩

다형성(Polymorphism)은 같은 이름의 메서드가 클래스에 따라 다르게 동작하는 것입니다. 메서드 오버라이딩(Overriding)을 통해 자식 클래스가 부모 클래스의 메서드를 재정의하여 구현합니다.

오버라이딩 비교

오버라이딩 전 - 부모 메서드만 사용
class Animal:
    def speak(self):
        print("동물이 소리를 냅니다.")

class Dog(Animal):
    pass  # speak() 미정의

class Cat(Animal):
    pass  # speak() 미정의

dog = Dog()
cat = Cat()
dog.speak()  # "동물이 소리를 냅니다."
cat.speak()  # "동물이 소리를 냅니다."
# 모두 동일한 출력 → 차별화 없음
오버라이딩 후 - 각 클래스가 다르게 동작
class Animal:
    def speak(self):
        print("동물이 소리를 냅니다.")

class Dog(Animal):
    def speak(self):  # 오버라이딩!
        print("멍멍!")

class Cat(Animal):
    def speak(self):  # 오버라이딩!
        print("야옹!")

animals = [Dog(), Cat(), Animal()]
for animal in animals:
    animal.speak()
# 멍멍!
# 야옹!
# 동물이 소리를 냅니다.

4 실습: 동물원 관리 시스템

학습 목표: 캡슐화·상속·다형성을 모두 활용하여 동물원 동물 관리 시스템을 설계하고 구현합니다.

전체 코드

Python 동물원 관리 시스템 
class Animal:
    def __init__(self, name, age, species):
        self.__name = name       # private
        self.__age = age         # private
        self.species = species   # public

    def get_name(self): return self.__name
    def get_age(self): return self.__age

    def speak(self):
        print(f"{self.__name}: 소리를 냅니다")

    def info(self):
        print(f"[{self.species}] {self.__name} ({self.__age}살)")

class Lion(Animal):
    def __init__(self, name, age):
        super().__init__(name, age, "사자")

    def speak(self):    # 오버라이딩
        print(f"{self.get_name()}: 으르렁! 🦁")

    def hunt(self):
        print(f"{self.get_name()}이(가) 사냥합니다!")

class Elephant(Animal):
    def __init__(self, name, age, weight):
        super().__init__(name, age, "코끼리")
        self.weight = weight

    def speak(self):    # 오버라이딩
        print(f"{self.get_name()}: 뿌우~ 🐘")

# 동물원 관리
zoo = [
    Lion("심바", 5),
    Elephant("덤보", 10, 3000),
    Lion("나나", 3)
]

print("=== 동물원 소개 ===")
for animal in zoo:
    animal.info()
    animal.speak()   # 다형성!
