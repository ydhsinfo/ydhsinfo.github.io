객체지향 프로그래밍의 이해

현실 세계의 사물을 객체로 추상화하고, 객체 간의 상호작용으로 프로그램을 구현하는 방법을 알아봅니다.

목록으로

1 객체지향 프로그래밍의 이해

객체지향 프로그래밍(OOP)은 현실 세계의 사물을 추상화하여 객체로 표현하고, 객체 간의 상호작용으로 프로그램을 구현하는 프로그래밍 패러다임입니다.

3가지 핵심 개념

객체

Object

고유한 속성과 기능을 가지는 데이터

클래스

Class

객체를 만들기 위한 설계도

인스턴스

Instance

클래스로 생성된 각각의 객체

1 객체 (Object)

객체란 고유한 속성(attribute)과 기능(method)을 가지는 데이터입니다. 우리 주변의 모든 사물을 객체로 볼 수 있습니다.

2 클래스 (Class)

클래스는 객체를 만들기 위한 설계도(blueprint)입니다. 클래스 안에는 속성(attribute)과 메소드(method)를 정의합니다.

설계도(Class) vs 실제 객체(Instance)

설계도 (Class)
Dice
value: int
roll() → random 1~6

"이렇게 만들어라" 라는 정의

실제 객체 (Instances)
dice1
⚀
value = 1
dice2
⚃
value = 4

설계도로 만들어진 실체들

dice_class.py 
import random

class Dice:
    def __init__(self):
        self.value = 0

    def roll(self):
        self.value = random.randint(1, 6)
        return self.value

3 인스턴스 (Instance)

인스턴스는 클래스로부터 생성된 각각의 객체입니다. 클래스를 한 번 정의하면, 이를 통해 많은 인스턴스를 생성할 수 있습니다.

자주 묻는 질문 (Q&A)

A 넓은 의미에서 객체와 인스턴스는 같은 것을 가리킵니다. 다만, "인스턴스"라는 용어는 클래스와 객체의 관계를 강조할 때 사용합니다.

객체

독립적으로 존재하는 사물 자체를 의미

인스턴스

"~클래스의 인스턴스"처럼 관계 중심 표현

예: "dice1은 Dice 클래스의 인스턴스이다" / "dice1은 객체이다"

A 클래스를 사용하면 코드의 재사용이 쉬워집니다. 설계도 하나로 동일한 구조의 객체를 빠르게 여러 개 만들 수 있고, 프로그램의 구조가 체계적이고 관리하기 쉬워집니다.

A 속성(attribute)은 객체의 데이터(상태)를 나타내고, 메소드(method)는 객체의 동작(기능)을 나타냅니다.

속성

주사위의 현재 값(value), 자동차의 색상, 학생의 이름

메소드

주사위 굴리기(roll()), 자동차 출발하기, 학생 공부하기

핵심 정리

객체(Object)

속성과 기능을 가진 데이터. 현실 세계의 사물을 추상화한 것.

클래스(Class)

객체의 설계도. 속성과 메소드를 정의하는 틀.

인스턴스(Instance)

클래스로 생성된 실제 객체. 하나의 클래스에서 여러 개 생성 가능.
