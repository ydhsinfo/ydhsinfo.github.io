원하는 데이터를 효율적으로 찾는 방법

여러 개의 데이터 중에서 원하는 데이터를 찾는 다양한 탐색 알고리즘의 원리를 이해하고, 직접 시뮬레이션하며 효율성을 비교합니다.

순차 탐색 이진 탐색 효율성 비교 그래프 탐색

학습 목표

  • 1 순차 탐색의 원리와 과정을 이해하고 설명할 수 있다.
  • 2 이진 탐색의 원리와 과정을 이해하고 설명할 수 있다.
  • 3 두 탐색 알고리즘의 효율성을 비교할 수 있다.

탐색이란?

탐색은 여러 개의 데이터 중에서 원하는 데이터를 찾는 과정입니다. 우리의 일상에서도 탐색은 매우 자주 사용됩니다.

1

순차 탐색 (Linear Search)

데이터를 처음부터 차례대로 비교하며 찾는 가장 단순한 탐색 방법

작동 원리

나열된 데이터를 처음부터 차례대로 비교하며 원하는 데이터를 찾는 방법입니다. 가장 단순한 탐색 알고리즘으로, 적은 양의 데이터를 탐색할 때 유리합니다.

예시: [10, 20, 30, 40, 50, 60, 70]에서 50 찾기

단계 1 10 ≠ 50 → 다음으로 이동
단계 2 20 ≠ 50 → 다음으로 이동
단계 3 30 ≠ 50 → 다음으로 이동
단계 4 40 ≠ 50 → 다음으로 이동
단계 5 50 = 50 → 탐색 종료! (5번 비교)

2

이진 탐색 (Binary Search)

정렬된 데이터에서 중앙값과 비교하여 탐색 범위를 절반씩 줄이는 방법

작동 원리

나열된 데이터의 중앙에 있는 데이터를 기준으로 찾는 데이터가 왼쪽인지 오른쪽인지 판단하며 탐색합니다. 데이터가 반드시 정렬되어 있어야 합니다.

L

왼쪽 위치

M

가운데 위치

M = (L+R)/2 (소수점 버림)

R

오른쪽 위치

예시: [10, 20, 30, 40, 50, 60, 70]에서 50 찾기

단계 1
L=0, R=6, M=3 → 중앙값=40
50 > 40 → 오른쪽 탐색
단계 2
L=4, R=6, M=5 → 중앙값=60
50 < 60 → 왼쪽 탐색
단계 3
L=4, R=4, M=4 → 중앙값=50
50 = 50 → 탐색 종료! (3번 비교)

주의! 이진 탐색은 데이터가 반드시 정렬(오름차순 또는 내림차순)되어 있어야 사용할 수 있습니다.

핵심 정리

순차 탐색

  • 데이터를 처음부터 차례대로 비교
  • 정렬이 필요 없음
  • 적은 양의 데이터에 적합
  • 최악의 경우 N번 비교

이진 탐색

  • 중앙값과 비교하여 범위를 절반씩 축소
  • 데이터가 반드시 정렬되어 있어야 함
  • 대량 데이터에서 매우 효율적
  • 최악의 경우 log2N번 비교

그래프 탐색

DFS (깊이 우선): 한 방향으로 깊이 탐색 - 경로 탐색에 유리
