알고리즘의 이해

문제를 효율적으로 해결하기 위한 단계적 절차인 알고리즘의 개념과 설계, 표현 방법, 그리고 효율성 분석을 학습합니다.

알고리즘 설계 알고리즘 표현 효율성 분석

학습 목표

  • 1 알고리즘의 개념과 세 가지 제어 구조를 이해할 수 있다.
  • 2 알고리즘을 자연어, 의사 코드, 순서도로 표현할 수 있다.
  • 3 알고리즘의 효율성을 비교하고 분석할 수 있다.

알고리즘이란?

알고리즘은 문제 해결을 위한 방법이나 절차를 의미합니다. 주어진 문제에 대해 입력을 받아 처리하고, 결과를 출력하는 일련의 과정입니다.

알고리즘 설계

세 가지 제어 구조

알고리즘은 세 가지 기본 제어 구조로 설계됩니다. 순서대로 실행하는 순차 구조, 조건에 따라 분기하는 선택 구조, 특정 조건 동안 반복하는 반복 구조입니다.

순차 구조

Sequential

위에서 아래로 순서대로 명령을 실행하는 구조입니다. 놀이공원 방문을 예로 들어보겠습니다.

선택 구조

Selection

조건에 따라 다른 명령을 실행하는 구조입니다. 키가 140cm 이상인지에 따라 롤러코스터 탑승 여부가 결정됩니다.

반복 구조

Iteration

특정 조건이 만족되는 동안 명령을 반복 실행하는 구조입니다. 대기줄에 사람이 있으면 기다리고, 없으면 놀이기구에 탑승합니다.

자연어 표현

말이나 글로 알고리즘을 표현하는 방법입니다.

  1. 1 나이를 입력받는다.
  2. 2 만약 나이가 9세 미만이면 입장료는 30,000원이다.
  3. 3 그렇지 않고 나이가 18세 미만이면 입장료는 40,000원이다.
  4. 4 그렇지 않으면 입장료는 50,000원이다.
  5. 5 입장료를 출력한다.

알고리즘의 효율성 분석

기억 장소와 수행 시간

효율적인 알고리즘이란 기억 장소 사용량이 적고, 수행 시간이 짧은 알고리즘입니다. 1부터 N까지의 합을 구하는 두 가지 방법을 비교해 봅시다.

1

방법 1: 순차적 덧셈

비효율적 
합 = 0
반복 i = 1 부터 N 까지
    합 = 합 + i
출력 합

덧셈 횟수: N번

수행 시간: N이 커질수록 증가

1 + 2 + 3 + ... + 100 = 100번의 덧셈
효율적
2

방법 2: 가우스 공식

효율적 
합 = N * (N + 1) / 2
출력 합

연산 횟수: 3번 (곱셈, 덧셈, 나눗셈)

수행 시간: N에 관계없이 일정

101 x 100 / 2 = 5050 (3번의 연산)

효율성 비교 계산기

핵심 정리

1 알고리즘 설계

  • 순차 구조: 순서대로 명령을 실행하는 구조
  • 선택 구조: 조건에 따라 다른 명령을 실행하는 구조
  • 반복 구조: 조건에 따라 반복 실행하는 구조

2 알고리즘의 표현

  • 자연어: 말이나 글로 알고리즘을 표현
  • 의사 코드: 프로그램 코드를 흉내 내어 표현
  • 순서도: 정해진 기호(타원, 직사각형, 마름모, 평행사변형, 화살표)로 표현

3 알고리즘의 효율성

  • 효율적인 알고리즘: 기억 장소 사용량이 적고, 수행 시간이 짧은 알고리즘
  • 같은 문제도 알고리즘에 따라 효율성이 크게 달라질 수 있음
