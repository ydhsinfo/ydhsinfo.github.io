정렬이란?

What is Sorting?

정렬의 정의

정렬(Sorting)은 수많은 데이터를 특정 기준에 따라 순서대로 나열하는 것을 말합니다. 정렬은 데이터를 효율적으로 검색하고 관리하기 위한 가장 기본적인 알고리즘 중 하나입니다.

오름차순

Ascending Order

작은 것에서 큰 것 순서로 나열합니다.

숫자 : 0 1 2 ... 9
영문 : A B ... Z
한글 : &HanGul; ㄴ ... ㅎ

내림차순

Descending Order

큰 것에서 작은 것 순서로 나열합니다.

숫자 : 9 8 ... 0
영문 : Z Y ... A
한글 : ㅎ ... ㄴ ㄱ

일상 속 정렬 활용 예시

쇼핑몰 앱

상품을 가격순으로 정렬하여 가장 저렴하거나 비싼 상품을 빠르게 찾을 수 있습니다.

낮은 가격순 / 높은 가격순

채팅 앱

채팅방을 최신 메시지순으로 정렬하여 가장 최근 대화를 먼저 확인할 수 있습니다.

최신순 정렬

놀이공원 앱

놀이기구를 대기시간순으로 정렬하여 대기 시간이 짧은 놀이기구를 빠르게 찾을 수 있습니다.

대기시간 짧은순

(1) 버블 정렬

Bubble Sort

버블 정렬의 원리

인접한 두 개의 데이터를 비교하여, 정렬 기준에 맞지 않으면 자리를 교환하는 방법입니다. 배열의 끝까지 이 과정을 반복하면 가장 큰 값이 맨 뒤로 이동합니다.

이름의 유래: 정렬 과정에서 데이터가 교환되는 모습이 마치 거품(bubble)이 수면 위로 올라오는 듯한 모습과 비슷하여 버블 정렬이라 합니다.

버블 정렬 시뮬레이터

데이터 입력:
비교 횟수: 0
교환 횟수: 0
현재 단계: 0

"초기화" 버튼을 누른 후 "다음 단계" 버튼으로 정렬 과정을 확인하세요.

버블 정렬 과정 예시: [5, 2, 7, 3] 오름차순 정렬

단계 1

5와 2 비교: 5 > 2 이므로 교환 [2, 5, 7, 3]

5와 7 비교: 5 < 7 이므로 유지 [2, 5, 7, 3]

7과 3 비교: 7 > 3 이므로 교환 [2, 5, 3, 7]

7이 확정 위치에 고정

단계 2

2와 5 비교: 2 < 5 이므로 유지 [2, 5, 3, 7]

5와 3 비교: 5 > 3 이므로 교환 [2, 3, 5, 7]

5가 확정 위치에 고정

단계 3

2와 3 비교: 2 < 3 이므로 유지 [2, 3, 5, 7]

3이 확정 위치에 고정

정렬 완료: [2, 3, 5, 7]

(2) 선택 정렬

Selection Sort

선택 정렬의 원리

첫 번째 위칫값을 기준으로 설정한 뒤, 나머지 데이터에서 가장 작은 값을 찾아 기준 위치의 값과 교환합니다. 기준 위치를 하나씩 뒤로 옮기며 이 과정을 반복합니다.

기준 위치
최솟값
정렬 완료

선택 정렬 시뮬레이터

데이터 입력:
비교 횟수: 0
교환 횟수: 0
현재 단계: 0

"초기화" 버튼을 누른 후 "다음 단계" 버튼으로 정렬 과정을 확인하세요.

선택 정렬 과정 예시: [5, 2, 7, 3] 오름차순 정렬

단계 1

기준: 5 (첫 번째 위치)

나머지 [2, 7, 3]에서 최솟값: 2

기준(5)과 최솟값(2) 교환 [2, 5, 7, 3]

2가 확정 위치에 고정

단계 2

기준: 5 (두 번째 위치)

나머지 [7, 3]에서 최솟값: 3

기준(5)과 최솟값(3) 교환 [2, 3, 7, 5]

3이 확정 위치에 고정

단계 3

기준: 7 (세 번째 위치)

나머지 [5]에서 최솟값: 5

기준(7)과 최솟값(5) 교환 [2, 3, 5, 7]

5가 확정 위치에 고정

정렬 완료: [2, 3, 5, 7]

Efficiency Comparison

핵심 결론

두 알고리즘의 비교 횟수는 동일하지만, 교환 횟수에서 차이가 납니다. 버블 정렬은 인접한 요소를 매번 교환하므로 교환 횟수가 많지만, 선택 정렬은 각 단계에서 최솟값을 찾은 후 한 번만 교환하므로 교환 횟수가 적어 더 효율적입니다.

핵심 정리

Key Summary

정렬

데이터를 특정 기준에 따라 순서대로 나열하는 것. 오름차순(작은 것 → 큰 것)과 내림차순(큰 것 → 작은 것)이 있습니다.

버블 정렬

인접한 두 데이터를 비교하여 자리를 교환하는 정렬 방법. 매 단계마다 가장 큰 값이 뒤쪽에 고정됩니다.

선택 정렬

기준 위치에서 나머지 데이터 중 최솟값을 찾아 교환하는 정렬 방법. 매 단계마다 가장 작은 값이 앞쪽에 고정됩니다.

효율성 비교

비교 횟수는 동일하지만, 교환 횟수에서 선택 정렬이 더 효율적입니다. 선택 정렬은 각 단계에서 한 번만 교환합니다.
