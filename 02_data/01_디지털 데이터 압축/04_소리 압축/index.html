소리 압축

아날로그 소리를 디지털로 변환하는 과정과 다양한 소리 압축 방식을 이해해 봅시다

WAV 파일 손실/비손실 압축 MP3 / AAC / FLAC

학습 목표

1 아날로그 소리를 디지털로 변환하는 과정(표본화, 양자화, 부호화)을 이해한다.
2 WAV 파일의 특성과 디지털 음원의 용량을 계산할 수 있다.
3 FLAC, MP3, AAC 등 다양한 소리 압축 방식의 특징을 비교한다.

WAV 파일

Waveform Audio File Format

WAV 파일이란?

WAV(Waveform Audio)는 웨이브 방식의 디지털 신호를 압축하지 않은 무압축 파일로, 확장자는 .wav입니다. 원본 소리 데이터를 그대로 저장하기 때문에 음질이 우수하지만 파일 크기가 큽니다.

아날로그 → 디지털 변환 과정

아날로그 소리 신호를 디지털로 변환하려면 다음 세 단계를 거칩니다.

1

표본화

Sampling

아날로그 소리 파형을 일정 간격으로 나누어 샘플 신호의 값을 측정합니다.

핵심 용어 주파수: 1초당 샘플링 횟수 (Hz)
2

양자화

Quantization

소리의 크기를 가까운 정숫값으로 표현합니다.

핵심 용어 진폭: 소리 신호의 크기(세기)
3

부호화

Encoding

각 정수를 0과 1의 디지털값으로 표현합니다.

핵심 용어 비트: 디지털 데이터의 최소 단위 (0 또는 1)
아날로그 신호 표본화 양자화 부호화 디지털 신호

소리 압축 파일

Audio Compression Formats

WAV 파일은 고음질이지만 용량이 매우 크기 때문에 다양한 압축 기술이 개발되었습니다. 소리 압축 방식은 비손실 압축과 손실 압축으로 나뉩니다.

FLAC

Free Lossless Audio Codec

비손실
  • 무음 구간과 반복 구간을 더 적은 비트로 표현
  • 원음 파형의 예측 수식과 실제 차이만 저장
  • 압축 해제 시 원본 복원 가능
파일 크기 예시 4.01 MB

MP3

MPEG-1 Audio Layer 3

손실
  • 사람이 잘 듣지 못하는 대역의 신호를 제거
  • 비트 전송률(bitrate)로 음질 조절 가능
  • 가장 널리 사용되는 음악 포맷
파일 크기 예시 361 KB

AAC

Advanced Audio Coding

손실
  • MP3와 같은 방식으로 압축
  • 동일 비트 전송률에서 MP3보다 음질이 더 좋음
  • Apple Music 등 스트리밍 서비스에서 활용
파일 크기 예시 366 KB

핵심 용어 정리

Key Terms

표본화 Sampling

아날로그 소리 파형을 일정 간격으로 나누어 신호의 값을 측정하는 과정

양자화 Quantization

측정된 소리의 크기를 가까운 정숫값으로 표현하는 과정

부호화 Encoding

각 정숫값을 0과 1의 이진수(디지털값)로 표현하는 과정

비트 전송률 Bitrate

1초당 전송되는 비트 수. 높을수록 음질이 좋지만 파일 크기가 커짐

주파수 Frequency

1초당 파형이 반복되는 횟수(Hz). 소리의 높낮이를 결정

진폭 Amplitude

소리 파형의 크기(세기). 진폭이 클수록 소리가 크게 들림

학습 정리

Summary

WAV 파일

  • 무압축 오디오 파일 (.wav 확장자)
  • 변환 과정: 표본화 → 양자화 → 부호화
  • CD 음질 1초 = 약 1,411,200 bit (≒ 0.16MB)

소리 압축 파일

  • FLAC: 비손실 압축, 원본 복원 가능
  • MP3: 손실 압축, 사람이 듣지 못하는 대역 제거
  • AAC: 손실 압축, MP3보다 음질 우수
