데이터 압축이란?

What is Data Compression?

데이터 압축의 정의

데이터 압축은 한정된 저장 공간에 데이터를 효율적으로 기록하기 위한 기술입니다. 원래의 데이터를 더 적은 비트(bit)로 표현하여 저장 공간을 절약하고, 네트워크 전송 시 속도를 향상시킵니다.

데이터 압축의 장점

저장 공간 절약

동일한 저장 장치에 더 많은 데이터를 보관할 수 있어 비용을 절감합니다.

전송 속도 향상

네트워크를 통해 데이터를 전송할 때 용량이 줄어들어 더 빠르게 전달됩니다.

디지털 탄소 중립

데이터 전송량 감소로 서버 부하와 에너지 소비를 줄여 탄소 중립 실천에 기여합니다.

인코딩과 디코딩

Encoding & Decoding

인코딩 (Encoding)

데이터 압축 과정

데이터를 더 적은 비트로 변환하는 과정입니다.

  • 원본 데이터에서 반복되는 패턴을 찾아 효율적으로 표현
  • 불필요하거나 중복된 정보를 제거하여 크기를 줄임

디코딩 (Decoding)

데이터 복원 과정

인코딩된 압축 데이터를 원래 형태로 복원하는 과정입니다.

  • 압축된 데이터를 원래의 형태로 되돌리는 역과정
  • 압축 방식에 따라 원본과 동일하거나 유사하게 복원

데이터 압축 처리 흐름

입력 원본 데이터
인코딩 압축 변환
저장 및 전송 압축 상태
디코딩 복원 변환
출력 복원 데이터
처리 흐름 설명

원본 데이터를 인코딩(압축)하여 저장하거나 전송한 뒤, 필요할 때 디코딩(복원)하여 사용합니다. 압축 방식에 따라 출력 데이터가 원본과 동일할 수도, 다를 수도 있습니다.

손실 압축 vs 비손실 압축

Lossy vs Lossless Compression

압축 방식은 데이터 유실 여부에 따라 비손실 압축과 손실 압축으로 나뉩니다.

비손실 압축

Lossless Compression

입력 데이터 = 출력 데이터

복원했을 때 입력 데이터와 출력 데이터가 같습니다. 원본 데이터를 완벽하게 되살릴 수 있습니다.

  • 원본 데이터의 완벽한 복원 가능
  • 정확성이 중요한 데이터에 적합
  • 압축률이 상대적으로 낮음
ZIP PNG FLAC 텍스트 파일

손실 압축

Lossy Compression

입력 데이터 ≠ 출력 데이터

복원했을 때 데이터가 달라집니다. 사람이 인지하기 어려운 일부 정보를 제거하여 높은 압축률을 달성합니다.

  • 높은 압축률로 파일 크기 대폭 감소
  • 멀티미디어(이미지, 음악, 동영상)에 적합
  • 원본 데이터의 완벽한 복원 불가능
JPEG MP3 MP4 H.264

손실 vs 비손실 압축 비교

구분 비손실 압축 손실 압축
복원 결과 원본과 동일하게 복원 원본과 다르게 복원 (일부 정보 손실)
압축률 상대적으로 낮음 상대적으로 높음
적합한 용도 텍스트, 소스 코드, 의료 이미지 등 정확성이 중요한 데이터 사진, 음악, 동영상 등 사람의 감각으로 인지하는 미디어
대표 형식 ZIP, PNG, FLAC, GIF JPEG, MP3, MP4, AAC
핵심 원리 반복 패턴 제거, 통계적 부호화 인간이 인지하기 어려운 정보 제거

쉽게 이해하기

비손실 압축은 책을 요약본으로 만든 뒤, 다시 원래의 책으로 완벽하게 복원하는 것과 같습니다. 반면 손실 압축은 그림을 축소해서 보관하는 것과 같아서, 다시 확대하면 원래보다 흐릿해질 수 있습니다.

핵심 정리

데이터 압축의 개념

  • 한정된 저장 공간에 데이터를 효율적으로 기록하는 기술
  • 인코딩(압축)과 디코딩(복원)의 과정으로 구성
  • 저장 공간 절약, 전송 속도 향상, 탄소 중립 실천

압축의 종류

  • 비손실 압축: 원본과 동일하게 복원 (ZIP, PNG, FLAC)
  • 손실 압축: 일부 정보 제거, 높은 압축률 (JPEG, MP3)
  • 용도에 따라 적절한 압축 방식을 선택하는 것이 중요
