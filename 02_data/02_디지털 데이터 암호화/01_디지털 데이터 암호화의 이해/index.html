디지털 데이터 암호화란?

Digital Data Encryption

핵심 개념

디지털 데이터 암호화란 권한이 없는 사람이 데이터를 알아볼 수 없게 변환하는 기술입니다. 인터넷 뱅킹, 메신저, 이메일 등 우리가 일상에서 사용하는 거의 모든 디지털 서비스에서 암호화 기술이 활용되고 있습니다.

암호화 (Encryption)

평문 → 암호문

평문(plaintext)을 암호문(ciphertext)으로 바꾸는 과정입니다.

LOVE 키=3 ORYH

복호화 (Decryption)

암호문 → 평문

암호문(ciphertext)을 다시 평문(plaintext)으로 복원하는 과정입니다.

ORYH 키=3 LOVE

암호화의 구성 요소

키 (Key)

암호 알고리즘 내에서 평문을 암호문으로 변경할 때 사용하는 문자열입니다. 올바른 복호화 키를 가진 사람만이 암호문을 해독할 수 있으므로, 키 관리가 매우 중요합니다.

암호 알고리즘

데이터를 암호화하고 복호화하는 수학적 절차입니다. 카이사르 암호, AES, RSA 등 다양한 알고리즘이 있으며, 알고리즘의 강도에 따라 보안 수준이 달라집니다.

키 관리의 중요성

아무리 강력한 암호 알고리즘을 사용하더라도, 키가 유출되면 암호문이 쉽게 해독될 수 있습니다. 따라서 키를 안전하게 생성, 저장, 전달, 폐기하는 것이 암호화에서 가장 중요한 과제입니다.

암호화 / 복호화 흐름

Encryption & Decryption Flow

평문이 키와 암호 알고리즘을 통해 암호문으로 변환되고, 올바른 키로 다시 평문으로 복원됩니다.

카이사르 암호 알고리즘

Caesar Cipher Algorithm

카이사르 암호란?

고대 로마의 줄리어스 카이사르(Julius Caesar)가 사용한 가장 기본적인 암호 방식입니다. 알파벳을 키(key) 값만큼 이동하여 다른 알파벳으로 치환하는 방식으로 동작합니다.

예시: 키(Key) = 3

각 알파벳을 3칸씩 뒤로 이동

변환 과정

대칭키 vs 공개키 암호화

Symmetric vs Asymmetric Encryption

대칭키 암호화

Symmetric Encryption

암호화 키
=
복호화 키
  • 암호화 키와 복호화 키가 동일합니다.
  • 처리 속도가 빠름 (대용량 데이터에 적합)
  • 키 전달 과정에서 유출 위험이 있음
  • 예: AES, DES, 카이사르 암호

공개키 암호화

Asymmetric Encryption

공개키
≠
개인키
  • 암호화 키와 복호화 키가 서로 다름
  • 키 전달 과정이 안전 (공개키만 공유)
  • 처리 속도가 느림 (복잡한 수학 연산)
  • 예: RSA, ECC, 전자서명

비교 요약표

구분 대칭키 암호화 공개키 암호화
키 구성 암호화 키 = 복호화 키 공개키 + 개인키 (한 쌍)
처리 속도 빠름 느림
키 전달 안전성 위험 안전
키 개수 (N명) N(N-1)/2 개 2N 개
대표 알고리즘 AES, DES, 카이사르 RSA, ECC
주요 용도 대용량 데이터 암호화 키 교환, 전자서명

Chapter Summary

암호화의 개념

  • 권한 없는 사람이 데이터를 알아볼 수 없게 변환하는 기술
  • 암호화(평문 → 암호문)와 복호화(암호문 → 평문)
  • 구성 요소: 키(key)와 암호 알고리즘

암호화 방식 비교

  • 대칭키: 같은 키로 암호화/복호화 (빠르지만 키 전달 위험)
  • 공개키: 다른 키로 암호화/복호화 (안전하지만 느림)
  • 카이사르 암호: 알파벳 이동 방식의 대칭키 암호
