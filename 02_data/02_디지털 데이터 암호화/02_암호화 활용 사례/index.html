1

고대 · 근대 암호화

적군으로부터 아군의 정보를 보호하기 위한 목적

전치 암호

평문의 문자를 자리 바꿔 암호화하는 방식입니다. 원래 문자는 그대로 사용하되, 순서를 재배열하여 의미를 숨깁니다.

핵심: 문자는 동일, 위치만 변경

치환 암호

평문의 문자를 다른 문자로 대체하여 암호화하는 방식입니다. 원래 문자를 규칙에 따라 전혀 다른 문자로 바꿉니다.

핵심: 위치는 동일, 문자를 대체

작동 원리

  1. 1 굵기와 길이가 같은 나무 막대기를 송신자와 수신자가 공유합니다.
  2. 2 양피지 리본을 막대기에 감은 뒤 옆으로 메시지를 작성합니다.
  3. 3 리본을 풀면 문자의 순서가 뒤섞여 원래 메시지를 알 수 없게 됩니다.
  4. 4 수신자가 같은 막대기에 감으면 원문을 복원할 수 있습니다.

암호화 예시

평문 Secret Message
암호문 (전치) Stse acMgreees

작동 원리

평문의 각 문자를 키(key)만큼 평행 이동한 문자로 치환합니다. 예를 들어, key=3이면 A는 D로, B는 E로, S는 V로 바뀝니다.

카이사르는 실제로 key=3을 사용하여 군사 기밀 통신에 활용했다고 전해집니다.

카이사르 암호 체험하기

암호문 결과:
VHFUHW

암호화 과정

1
키보드 입력

평문 문자를 키보드로 입력

2
배전반 1차 치환

배전반에서 문자를 1차로 변환

3
회전자 2차 치환

회전자를 통과하며 추가 변환

4
발광판 출력

암호화된 문자가 발광판에 표시

경우의 수

회전자 1개로도 경우의 수가 676개 (26 x 26)에 달합니다. 에니그마는 여러 개의 회전자와 배전반을 조합하여 사실상 해독이 불가능에 가까운 수준의 암호화를 구현했습니다.

2

현대 암호화

디지털 시대의 정보 보호 기술

해시 함수란?

  • 별도의 키 없이 일정 규칙을 적용하여 고정된 길이의 해시값을 출력
  • 암호화만 가능하고, 복원(복호화)은 불가능 (일방향 함수)
  • 패스워드가 유출되어도 원본 비밀번호를 복원할 수 없음
  • 로그인 시 입력값을 해시화하여 저장된 해시값과 비교

보안 핵심 포인트

해시 함수는 암호화만 가능하고 복원이 불가능하기 때문에, 패스워드가 유출되어도 원래 비밀번호를 알아낼 수 없습니다. 로그인 시에는 입력한 비밀번호를 다시 해시화하여 저장된 해시값과 비교합니다.

디지털 서명 과정

송신자 (서명 생성)
  1. 1 원본 메시지에 해시 함수를 적용하여 해시값 생성
  2. 2 송신자의 개인키(Private Key)로 해시값을 암호화
  3. 3 암호화된 해시값 = 디지털 서명
수신자 (서명 검증)
  1. 1 송신자의 공개키(Public Key)로 디지털 서명을 복호화
  2. 2 수신한 메시지에 같은 해시 함수를 적용하여 해시값 비교
  3. 3 두 해시값이 일치하면 송신자 신원 확인 완료
🔑 Private Key ✍ 디지털 서명 전송 수신자 디지털 서명 공개키 복호화 🔓 Public Key 해시값 비교 ✓ ✅ 신원 확인 완료

HTTPS 작동 원리

1
공개키 암호화로 인증기관 확인

웹 사이트의 SSL 인증서를 인증기관(CA)의 공개키로 검증

2
대칭키 암호화로 데이터 전송

인증 완료 후, 대칭키를 공유하여 빠르고 안전하게 데이터 전송

사용자 확인 방법

https:// example.com
  • 사이트 주소가 'https'로 시작하는지 확인
  • 브라우저 주소창에 자물쇠 아이콘이 표시되는지 확인
  • 자물쇠 클릭 시 SSL 인증서 정보를 확인 가능

HTTPS에서 사용되는 암호화 방식 비교

구분 공개키 암호화 대칭키 암호화
용도 인증기관 확인 (신뢰성) 데이터 전송 (속도)
키 공개키 + 개인키 (2개) 동일한 키 1개 공유
특징 보안성 높음, 속도 느림 속도 빠름, 키 배포 문제

학습 내용 정리

고대 · 근대 암호화

  • 전치 암호 - 문자의 위치를 재배열 (스키테일)
  • 치환 암호 - 문자를 다른 문자로 대체 (카이사르)
  • 에니그마 - 기계식 다중 치환 (배전반 + 회전자)

현대 암호화

  • 패스워드 - 해시 함수로 일방향 암호화
  • 디지털 서명 - 개인키로 서명, 공개키로 검증
  • HTTPS - 공개키(인증) + 대칭키(전송) 조합
