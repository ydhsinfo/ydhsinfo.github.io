구글 시트 연동 가이드

학생 활동 데이터를 구글 시트에 자동으로 수집하는 방법을 안내합니다.

작동 원리


학생
시트 URL 입력 + 제출

프록시 스크립트
데이터 중계 (1회 설정)

교사 구글 시트
각 교사별 개별 시트

핵심: 사이트 관리자가 프록시 스크립트를 1회 배포하면, 다른 교사들은 구글 시트만 만들어서 학생에게 링크를 공유하면 됩니다.

🎉

코딩 없이 3분 만에 설정!

구글 시트를 만들고 → 공유 설정하고 → 학생에게 링크를 알려주면 끝!

1

구글 시트 만들기

학생 데이터를 받을 스프레드시트를 생성합니다

  1. 1. Google Sheets 에 접속합니다.
  2. 2. + 새 스프레드시트를 클릭합니다.
  3. 3. 시트 이름을 "1학년 정보 응답" 등으로 변경합니다.

헤더 자동 생성!

헤더(제출시간, 학년, 반 등)를 직접 입력할 필요가 없습니다.
학생이 처음 제출하면 페이지별로 시트 탭이 자동 생성되고 헤더도 자동으로 입력됩니다.

탭 1 네트워크 선택활동 → 미러링, 무선전송, 클라우드 데이터
탭 2 IoT 선택활동 → 코딩 실습 코드, 자기평가
탭 3 대단원 마무리 → 문제 1~6 답안, 자기평가
2

공유 설정하기

학생들의 데이터가 시트에 기록될 수 있도록 편집 권한을 설정합니다

  1. 1. 시트 오른쪽 상단의 공유 버튼을 클릭합니다.
  2. 2. "일반 액세스" 항목에서 "링크가 있는 모든 사용자"로 변경합니다.
  3. 3. 역할을 "편집자"로 설정합니다.
  4. 4. "링크 복사"를 클릭합니다.
중요: 반드시 "편집자" 권한이어야 합니다. "뷰어"로 설정하면 학생 데이터가 기록되지 않습니다.
3

학생에게 링크 공유

복사한 구글 시트 링크를 학생들에게 안내합니다

복사한 구글 시트 URL을 학생들에게 전달합니다. 전달 방법은 자유입니다:

칠판에 게시

학급 채팅방

QR 코드

e학습터

학생이 할 일:

  1. 1. 선택활동 웹페이지를 열고
  2. 2. "구글 시트 URL" 칸에 선생님이 공유한 링크를 붙여넣고
  3. 3. 학년, 반, 번호, 이름을 입력하고
  4. 4. 활동을 작성한 후 "제출하기" 버튼 클릭!

끝! 이게 전부입니다 🎉

학생이 제출하면 선생님의 구글 시트에 자동으로 데이터가 쌓입니다.
시트를 열어서 실시간으로 확인하세요!

사이트 관리자(개발자)가 프록시 Apps Script를 1회 배포하면, 이 사이트를 사용하는 모든 교사가 코딩 없이 구글 시트 연동을 사용할 수 있습니다.

예상 소요 시간: 약 10분 (1회)
1

Apps Script 프로젝트 생성

프록시 역할을 할 스크립트를 만듭니다

  1. 1. Google Apps Script 에 접속합니다.
  2. 2. + 새 프로젝트를 클릭합니다.
  3. 3. 기존 코드를 모두 지우고, 아래 코드를 복사하여 붙여넣습니다:
function doPost(e) { try { var data = JSON.parse(e.postData.contents); var sheetUrl = data.sheetUrl; // 구글 시트 URL에서 ID 추출 var match = sheetUrl.match(/\/spreadsheets\/d\/([a-zA-Z0-9-_]+)/); if (!match) { return error("올바른 구글 시트 URL이 아닙니다."); } var ss = SpreadsheetApp.openById(match[1]); var pageName = data.page || "제출"; // 페이지 이름으로 시트 탭 찾기 (없으면 자동 생성) var sheet = ss.getSheetByName(pageName); if (!sheet) { sheet = ss.insertSheet(pageName); var headers = getHeaders(pageName); sheet.getRange(1, 1, 1, headers.length) .setValues([headers]); sheet.getRange(1, 1, 1, headers.length) .setFontWeight("bold"); sheet.setFrozenRows(1); } // 페이지별 데이터 행 생성 후 추가 var row = buildRow(pageName, data); sheet.appendRow(row); return ContentService .createTextOutput(JSON.stringify({ result: "success" })) .setMimeType(ContentService.MimeType.JSON); } catch (err) { return ContentService .createTextOutput(JSON.stringify({ result: "error", message: err.message })) .setMimeType(ContentService.MimeType.JSON); } } // ====== 페이지별 헤더 정의 ====== function getHeaders(page) { var common = ["제출시간","학년","반","번호","이름"]; if (page === "네트워크 선택활동") { return common.concat([ "활동1-1기기","활동1-1방법","활동1-1모드", "활동1-2기기","활동1-2방법","활동1-2모드", "활동2-1방법","활동2-1장점","활동2-1단점", "활동2-2방법","활동2-2장점","활동2-2단점", "활동2-3방법","활동2-3장점","활동2-3단점", "활동3-1서비스","활동3-1방법","활동3-1파일", "활동3-2서비스","활동3-2방법","활동3-2파일", "자기평가" ]); } if (page === "IoT 선택활동") { return common.concat([ "가-인라인코드","가-전체코드", "나-인라인코드","나-전체코드", "자기평가1","자기평가2" ]); } if (page === "대단원 마무리") { return common.concat([ "문제1","문제2","문제3(주관식)", "문제4","문제5","문제6(서술형)", "자기평가1","자기평가2" ]); } // 기본: 모든 데이터를 JSON으로 저장 return common.concat(["데이터"]); } // ====== 페이지별 행 데이터 생성 ====== function buildRow(page, data) { var base = [ new Date(), data.grade, data["class"], data.number, data.name ]; if (page === "네트워크 선택활동") { return base.concat([ data.act1_row1_device, data.act1_row1_method, data.act1_row1_mode, data.act1_row2_device, data.act1_row2_method, data.act1_row2_mode, data.act2_row1_method, data.act2_row1_pros, data.act2_row1_cons, data.act2_row2_method, data.act2_row2_pros, data.act2_row2_cons, data.act2_row3_method, data.act2_row3_pros, data.act2_row3_cons, data.act3_row1_service, data.act3_row1_method, data.act3_row1_file, data.act3_row2_service, data.act3_row2_method, data.act3_row2_file, data.selfEval ]); } if (page === "IoT 선택활동") { return base.concat([ data.code_ga_inline, data.code_ga, data.code_na_inline, data.code_na, data.eval1, data.eval2 ]); } if (page === "대단원 마무리") { return base.concat([ data.q1_answer, data.q2_answer, data.q3_answer, data.q4_answer, data.q5_answer, data.q6_answer, data.eval1, data.eval2 ]); } // 기본: JSON으로 저장 return base.concat([JSON.stringify(data)]); }
  1. 4. Ctrl + S 로 저장합니다. 프로젝트 이름은 "학생 응답 프록시" 등으로 입력합니다.
2

웹 앱으로 배포

스크립트를 웹에서 접근 가능하도록 배포합니다

  1. 1. Apps Script 화면 오른쪽 상단의 [배포] → [새 배포]를 클릭합니다.
  2. 2. 유형 선택에서 [웹 앱]을 선택합니다.
  3. 3. 다음과 같이 설정합니다:
    실행 주체 나 (본인 계정)
    액세스 권한 모든 사용자
  4. 4. [배포] 버튼을 클릭합니다.
  5. 5. 권한 승인 요청이 나타나면 [액세스 허용]을 클릭합니다.
    "이 앱은 확인되지 않았습니다" 경고가 나타나면 [고급] → [프로젝트로 이동]을 클릭하면 됩니다.
  6. 6. 배포 완료 후 나타나는 웹 앱 URL을 복사합니다.
3

프록시 URL을 코드에 적용

배포한 URL을 선택활동 페이지에 입력합니다

  1. 1. 제출 기능이 있는 HTML 파일들에서 <script> 태그 안의 아래 줄을 찾습니다:
    const PROXY_SCRIPT_URL = '';
  2. 2. 빈 따옴표 안에 복사한 URL을 붙여넣습니다:
    const PROXY_SCRIPT_URL = 'https://script.google.com/macros/s/여기에URL/exec';
  3. 3. 적용 대상 파일:
    01_네트워크의 구성/03_선택활동/index.html
    02_사물 인터넷/03_선택활동/index.html
    03_대단원 마무리/index.html
  4. 4. 파일을 저장하고 GitHub에 푸시합니다.

관리자 설정 완료!

이제 이 사이트를 사용하는 모든 교사가
구글 시트만 만들면 제출 기능을 바로 사용할 수 있습니다.

주의사항

프록시 스크립트는 관리자의 Google 계정 권한으로 실행됩니다. 교사가 공유한 구글 시트가 "편집자" 권한으로 열려있어야 데이터를 기록할 수 있습니다.

Apps Script에는 일일 사용 제한이 있습니다. 일반적인 학교 규모에서는 충분합니다.

스크립트 코드를 수정했다면 [배포] → [배포 관리] → [수정]에서 버전을 "새 버전"으로 선택하고 다시 배포해야 합니다.

문제 해결

Q. 학생이 제출했는데 시트에 데이터가 안 들어와요

구글 시트의 공유 설정이 "링크가 있는 모든 사용자 - 편집자"로 되어 있는지 확인하세요.

Q. "올바른 구글 시트 URL이 아닙니다" 오류가 발생해요

학생이 입력한 URL이 https://docs.google.com/spreadsheets/d/... 형식인지 확인하세요. 구글 시트를 열고 주소창의 URL을 전체 복사하면 됩니다.

Q. "제출 실패" 오류가 계속 발생해요

사이트 관리자에게 프록시 스크립트가 정상 배포되어 있는지 확인을 요청하세요.

Q. 반별로 다른 시트에 데이터를 받을 수 있나요?

네! 반별로 서로 다른 구글 시트를 만들고, 각 반 학생에게 해당 시트 URL을 공유하면 됩니다.
